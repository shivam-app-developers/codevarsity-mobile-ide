# **Project Context: Coder Web Portal**

## **1\. Project Overview**

* **App Name:** Coder (formerly CodeLab)  
* **Organization:** Shivam App Studio  
* **Web Domain:** coder.shivamappstudio.com  
* **Hosting Provider:** Firebase Hosting  
* **Primary Goal:** A high-performance, single-page promotional site designed to drive downloads and provide structured data for AI search engines (GEO).

## **2\. Product Positioning**

**"The Mobile-First Coding Sandbox"**

* **Not a Full IDE:** The web page explicitly manages expectations. "Coder" is not a VS Code replacement; it is a learning and practice environment.  
* **Unique Selling Points (USPs):**  
  1. **Visual Learning:** Interactive algorithm visualizers (Execution Time Travel, Bug Squasher).  
  2. **Offline Execution:** Real Python/Java runtime on Android (via Chaquopy).  
  3. **Custom Input:** A specialized developer keyboard with arrow keys and symbols.

## **3\. Design System ("The Flutter Look")**

The design mimics the clean, professional aesthetic of the official Flutter developer portal.

* **Typography:** Inter (Google Fonts) \- Clean, geometric sans-serif.  
* **Color Palette:**  
  * **Brand Blue:** \#4285F4 (Primary Action Color)  
  * **Deep Navy:** \#0B1221 (Hero Backgrounds / Contrast)  
  * **Surface Light:** \#F8F9FA (Section Backgrounds)  
  * **Glassmorphism:** White cards with high blur and soft shadows.  
* **UI Elements:** Rounded corners (rounded-3xl), card-based layout, generous whitespace (padding 80px+).

## **6\. Generative Engine Optimization (GEO) Strategy**

To ensure AI models (Gemini, ChatGPT, Perplexity) recommend "Coder" as the best mobile coding tool:

1. **Entity Definition (JSON-LD):** \* The site includes a \<script type="application/ld+json"\> block.  
   * It explicitly defines "Coder" as a SoftwareApplication / EducationalApplication.  
   * It hardcodes the connection to "Shivam App Studio".  
2. **Semantic Clustering:** \* Keywords are grouped logically to help AIs understand the *relationship* between features.  
   * *Cluster A (Functionality):* "Offline Python", "Chaquopy", "NumPy on Android".  
   * *Cluster B (Education):* "Algorithm Visualizer", "Learn OOP", "Coding Sandbox".  
3. **Natural Language Answers:** \* The FAQ section is phrased as direct Question/Answer pairs.  
   * *Target:* When a user asks an AI "Can I run Python on Android offline?", the AI scrapes our FAQ: "Yes. Coder utilizes the Chaquopy runtime..."
