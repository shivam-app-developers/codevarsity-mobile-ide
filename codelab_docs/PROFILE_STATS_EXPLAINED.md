# Understanding Your CoderKit Profile

Your CoderKit profile showcases **verified coding activity** ‚Äî not just courses completed, but real skills demonstrated through practice and problem-solving.

## üèÜ Learning Stats

These metrics track your educational progress and time invested.

### Courses

**What it measures:** Complete learning paths finished on CoderKit.

Each course contains multiple modules with interactive lessons, coding exercises, and assessments. Completing a course means you've mastered all concepts within that track.

### XP (Experience Points)

**What it measures:** Cumulative learning activity across all interactions.

XP is earned through:

- Completing lessons
- Solving challenges
- Maintaining streaks
- Achieving milestones

Higher XP indicates consistent, long-term engagement with the platform.

### Lines Typed

**What it measures:** Total lines of code you've written in the editor.

**Why it matters:** This stat only counts code typed through CoderKit's custom keyboard ‚Äî not pasted text. It represents genuine coding practice, building muscle memory and syntax familiarity.

## üîß Problem Solving Stats

These metrics demonstrate your debugging and code improvement abilities ‚Äî the skills recruiters value most.

### Bug Squasher

**What it measures:** Debugging challenges completed.

In Bug Squasher challenges, you're given broken code and must identify and fix the errors to make it run correctly. This directly mirrors real-world debugging scenarios.

**Difficulty Breakdown:**

- **Easy:** Syntax errors, typos, simple logic issues
- **Medium:** Multi-line bugs, scope issues, logic errors
- **Hard:** Complex algorithms, edge cases, performance bugs

### Code Refactor

**What it measures:** Code improvement challenges completed.

Code Refactor challenges present inefficient or poorly-structured code. Your task is to rewrite it following best practices ‚Äî improving readability, performance, or maintainability.

This demonstrates your understanding of:

- Clean code principles
- Optimization techniques
- Software design patterns

### Errors Fixed

**What it measures:** Total syntax and runtime errors you've resolved in the IDE.

Every time you write code with errors and then correct them, the system tracks it. This shows your ability to identify and resolve issues independently ‚Äî a critical developer skill.

## üìÅ Building Stats

These metrics showcase your project creation and completion capabilities.

### Projects

**What it measures:** Independent coding projects created in CoderKit.

Projects are self-directed ‚Äî you choose what to build. This demonstrates initiative, creativity, and the ability to apply learned concepts to real applications.

### Capstones

**What it measures:** Guided capstone projects completed.

Capstone projects are comprehensive, multi-module projects that combine multiple skills. Completing them shows you can handle larger-scale development work.

### Scrambles

**What it measures:** Code Scramble puzzles solved.

Code Scramble challenges test your understanding of code structure. You're given jumbled code blocks and must arrange them in the correct order to form working programs.

This develops:

- Code reading comprehension
- Understanding of execution flow
- Pattern recognition

## üî• Consistency Stats

These metrics demonstrate discipline, persistence, and commitment to continuous learning.

### Streak

**What it measures:** Consecutive days of coding activity.

Your streak counts how many days in a row you've practiced on CoderKit. Even small daily practice compounds into significant skill growth over time.

**Why recruiters care:** A long streak shows discipline and commitment ‚Äî qualities that translate directly to workplace reliability.

### Practice

**What it measures:** Guided Practice exercises completed.

Guided Practice sessions help you learn by typing code examples character-by-character, building muscle memory and syntax familiarity through repetition.

### First-Try Rate

**What it measures:** Percentage of challenges you pass on your first attempt.

```
First-Try Rate = (Challenges Passed First Try / Total Challenges Attempted) √ó 100
```

**Why this matters:** This is a quality metric, not quantity. A high first-try rate indicates:

- Strong problem analysis before coding
- Careful, deliberate approach
- Solid foundational understanding

A 70%+ first-try rate is excellent and demonstrates genuine competence.

## ‚úì Verified & Authentic

### How We Ensure Authenticity

Your CoderKit stats are **verified** ‚Äî meaning they reflect genuine coding activity.

**Anti-Cheat Measures:**

- **Custom Keyboard Only:** In challenges, code must be typed through our custom keyboard ‚Äî no copy-paste from external sources
- **Activity Tracking:** We track when and how code is written, not just the final result
- **First-Try Tracking:** Your initial attempt is recorded, preventing brute-force solutions

**What "Verified by CoderKit" means:**

- Stats cannot be falsified or inflated
- Activity is tracked in real-time
- Profile data is synced to secure servers

### For Recruiters

When you see a CoderKit profile, you're seeing:

- **Actual skills demonstrated**, not just courses watched
- **Quality metrics** (first-try rate), not just quantity
- **Consistency evidence** (streaks), showing work ethic
- **Difficulty progression**, showing growth beyond basics
